### 4. 关键决策点（⭐ 评分重点）
- **金额单位**。合同模板正文一度让人以为按「元」。核对合同 §1「金额单位为分(int64)」+ fixtures（`current_price:8800`）后，确定表单按「元」收集、提交 `×100` 转分、展示 `÷100`，并在 `AuctionForm`/`format.ts` 注释固化，避免与后端差 100 倍。
- **AntD 版本**。模板正文写「AntD 5」，但 `package.json`/任务单 §0.1 实际是 AntD 6 → 以实际依赖为准用 v6，并在交付说明里点名该不一致。
- **角色鉴权边界**。`LoginData` 不含 role 字段；按 `dev-setup §5` 用 token 段（`-seller-`/`-user-`）做**前端体验分流**，并明确「鉴权以后端 DB 为唯一事实来源」，不在前端发明业务规则。
- **倒计时实现**。坚持 `requestAnimationFrame + server_time 偏移`（`time.ts`/`Countdown`），不用 `setInterval(1000)`，规避漂移（合同/反模式明确要求）。
- **lint 规则取舍**。react-hooks v7 的 `set-state-in-effect` 对「挂载即拉取」误报；对 4 处数据拉取加**定点** `eslint-disable` + 理由注释，而非全局关规则。
- **依赖冲突处理**。WSL 的 `HTTP_PROXY=http://:7897`（缺主机）导致 npm `ERR_INVALID_URL` → 安装时显式 unset 代理；`openapi-typescript@7` peer 要 TS5、本项目 TS6 → `.npmrc` 固化 `legacy-peer-deps`（仅 CLI 生成类型，运行期无关）。

### 5. 不适合交给 AI 的部分
- 跨人接口契约（openapi/合同）：只读不改，任何字段变更须走 `integration-protocol §2` 三人评审。
- 真实出价的事务一致性 / Redis 锁 / MySQL 条件更新：属 Role A 后端，前端只按 HTTP 响应渲染，不臆造乐观更新。
- 切真接口的联调时机与撤 mock：需人工对照 §3/§4 checklist 与对端确认。
