## 2. 高质量产出的 prompt 模板沉淀
- 见 `docs/prompts/role-C-merchant-ai-polish.md`（必读上下文 + 分场景模板 + 反模式）。
- 经验：**先喂合同/openapi/任务单再让 AI 动手**，比直接描述需求产出对齐度高得多。

## 3. 关键决策点合集（人工把控证据）
> 从各日志 §4 汇总。以下为已记录的代表性条目：
- 金额单位「分 vs 元」纠偏（差 100 倍的隐患）。
- AntD 5/6 文档不一致 → 以实际依赖为准。
- 角色鉴权只做前端分流，事实源在后端 DB。
- 倒计时用 rAF + server_time 偏移，拒绝 setInterval 漂移。
- lint 规则定点豁免而非全局关闭。

## 4. 不适合 AI 的场景
- 复杂事务一致性（出价的 Redis 锁 + MySQL 条件更新 + outbox）——必须人工设计与验证。
- 跨人接口契约变更——须走三人评审流程。
- 安全边界（token/鉴权）——mock 仅用于答辩，生产替换 JWT，决策不外包。
